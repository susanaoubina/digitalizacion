02

Sistemas operativos: el software que manda CA1.2

El sistema operativo (SO) es el programa principal que arranca al encender: gestiona el hardware, ejecuta las aplicaciones y te ofrece una interfaz. Sin SO, un ordenador es solo metal y silicio.

¿Qué es y qué hace?
Tipos de SO
Instalación
Configuración y archivos
🧩 Funciones del SO
  • Gestión del hardware: coordina CPU, memoria, discos y periféricos mediante controladores (drivers).
  • Gestión de procesos: decide qué programa usa la CPU y cuándo (multitarea).
  • Gestión de memoria: reparte la RAM entre las apps abiertas.
  • Sistema de archivos: organiza los datos en carpetas y ficheros.
  • Interfaz de usuario: escritorio, ventanas y menús (GUI) o línea de comandos (CLI).
  • Seguridad: usuarios, permisos y contraseñas.
🔍 ¿Dónde vive el SO?

Cuando pulsas el botón de encendido, la placa base ejecuta la BIOS/UEFI, un pequeño programa grabado en un chip. Este comprueba el hardware (POST) y luego busca el SO en el disco y lo carga en la RAM. A ese proceso se le llama arranque (boot).

Un mismo equipo puede tener varios SO (arranque dual) y elegir cuál usar al encender.

💻 Ordenadores de escritorio y portátiles
  • Windows — el más extendido en aulas y hogares. Propietario.
  • macOS — exclusivo de equipos Apple. Propietario.
  • GNU/Linux — libre y gratuito (Ubuntu, Debian, Mint…). Muy usado en servidores y educación.
📱 Dispositivos móviles
  • Android — basado en Linux, código abierto, de Google.
  • iOS / iPadOS — de Apple, propietario.

Libre vs. propietario: el software libre permite usarlo, estudiarlo, copiarlo y modificarlo; el propietario restringe estos derechos con una licencia.

1 · Preparar el medio de instalación ▶

Se descarga la imagen ISO del SO desde la web oficial y se crea un USB de arranque (booteable). Antes: hacer copia de seguridad de tus datos.

2 · Configurar el arranque en la UEFI/BIOS ▶

Al encender se pulsa una tecla (F2, F12, Supr…) para entrar en la UEFI y elegir arrancar desde el USB en lugar del disco.

3 · Particionar y copiar el sistema ▶

El instalador prepara el disco (particiones) y copia los archivos del SO. Aquí se decide si se conserva otro SO (arranque dual).

4 · Configuración inicial de usuario ▶

Idioma, zona horaria, teclado, nombre de usuario y contraseña, y conexión a la red. El SO instala los drivers básicos.

5 · Actualizar y añadir software ▶

Primer paso tras instalar: actualizaciones de seguridad. Después se instalan las aplicaciones de uso habitual (ofimática, navegador, editores…).

⚙️ Configuración de usuario
  • Cuentas y perfiles: cada persona con su usuario y permisos.
  • Personalización: fondo, resolución, accesibilidad (tamaño de texto, contraste, lector de pantalla).
  • Actualizaciones automáticas y copias de seguridad.
  • Gestión de aplicaciones: instalar, actualizar y desinstalar.
🗂️ Organización y almacenamiento
  • Jerarquía de carpetas: una estructura lógica (por asignatura, por proyecto…) ahorra tiempo.
  • Nombres consistentes: 2026-09_apuntes_redes.pdf se ordena solo.
  • Extensiones: .pdf .jpg .mp4 .docx indican el tipo de archivo.
  • Local vs. nube: la nube facilita el acceso y la copia de seguridad; lo local no depende de Internet.

🔌 Dispositivos de red
  • Router: dirige el tráfico entre tu red local e Internet; es la puerta de enlace.
  • Switch: conecta varios equipos dentro de la red local por cable.
  • Punto de acceso (AP): da cobertura Wi-Fi.
  • Tarjeta de red: en cada dispositivo, permite conectarse (Ethernet o Wi-Fi).
🏷️ Direcciones e identificadores
  • IP: la "dirección postal" de cada dispositivo en la red (ej. 192.168.1.10).
  • MAC: identificador físico único de cada tarjeta de red.
  • SSID: el nombre de tu red Wi-Fi.
  • DNS: traduce nombres (www.edu.xunta.gal) a direcciones IP.
🛠️ Configurar una red doméstica — pasos clave
1 Conectar el router y encenderlo.
2 Acceder al panel del router por el navegador.
3 Cambiar el nombre (SSID) y la contraseña Wi-Fi.
4 Elegir cifrado WPA2/WPA3.
5 Conectar dispositivos y comprobar la conexión.

📡 Con fíos (Ethernet): más estable y rápida, ideal fija. Sin fíos (Wi-Fi): cómoda y móvil, pero sensible a distancia e interferencias.

05

Internet de las cosas (IoT) y wearables

El IoT conecta a Internet objetos cotidianos que antes no lo estaban: recogen datos con sensores, los envían a la red y actúan según lo que reciben. Los wearables son IoT que llevamos puesto.

⌚
Smartwatch

Mide pulso, pasos y sueño; avisa de notificaciones.

💡
Bombilla smart

Se enciende y regula desde el móvil o por voz.

🔊
Altavoz de voz

Asistente que responde y controla otros dispositivos.

🌡️
Termostato

Ajusta la calefacción según hábitos y ahorra energía.

📷
Cámara / timbre

Vigilancia y aviso al móvil en tiempo real.

🚗
Coche conectado

Navegación, diagnóstico y actualizaciones OTA.

🔗 Cómo se conecta un dispositivo IoT
  • Se instala la app del fabricante.
  • El dispositivo crea una red temporal o usa Bluetooth para el emparejamiento.
  • Se le da acceso a tu Wi-Fi y se vincula a tu cuenta.
  • Se configuran permisos, automatizaciones y actualizaciones.
⚠️ Lo que hay que vigilar
  • Privacidad: recogen muchos datos personales (ubicación, salud, audio).
  • Seguridad: cambia siempre la contraseña por defecto y actualiza el firmware.
  • Dependencia: si falla la nube del fabricante, el dispositivo puede dejar de funcionar.

Uso responsable: reutilización y ahorro energético

El currículo pide crear hábitos de reutilización de materiales y ahorro energético. La tecnología tiene un coste ambiental: fabricar, alimentar y desechar dispositivos.

♻️ Alargar la vida útil
  • Ampliar RAM o cambiar a SSD revive equipos "lentos".
  • Instalar un SO ligero (Linux) en máquinas antiguas.
  • Reparar antes que sustituir; donar o reciclar en puntos limpios (RAEE).
🔋 Reducir el consumo
  • Modos de ahorro de energía y suspensión.
  • Apagar en lugar de dejar en espera prolongada.
  • Brillo, cierre de apps en segundo plano y desconexión de periféricos.
  • Cuidado con la obsolescencia programada: elige productos reparables.

Glosario rápido

Hardware

Componentes físicos de un dispositivo.

Software

Programas y datos; las instrucciones que ejecuta el hardware.

CPU

Unidad central de proceso; ejecuta las instrucciones.

RAM

Memoria temporal y rápida donde trabaja el equipo.

SSD / HDD

Almacenamiento permanente de datos.

Sistema operativo

Software base que gestiona el hardware y las apps.

Driver

Programa que permite al SO comunicarse con un componente.

UEFI/BIOS

Firmware que arranca el equipo antes del SO.

Router

Dispositivo que conecta tu red con Internet.

IP

Dirección de un dispositivo dentro de una red.

SSID

Nombre de una red Wi-Fi.

IoT

Objetos cotidianos conectados a Internet.
